源码实现：合并渲染架构，得到可用的 render 函数 ​

现在还存在一个问题，那就是我们应该 如何使用 render 函数呢？

在源码中，我们可以直接：

javascript
const { render } = Vue

render(vnode, document.querySelector('#app'))

但是，现在在我们自己的实现当中，我们无法导出当前的render函数，那么 vue 是怎么做的呢？

查看源码 ​

查看源代码，我们可以发现在 packages/runtime-dom/src/index.ts 中，存在这样的一段代码：

javascript
export const render = ((...args) => {
  ensureRenderer().render(...args)
}) as RootRenderFunction<Element | ShadowRoot>

而这里导出的 render 就是我们 解构出来的 render。

在这段代码中涉及到一个方法 ensureRenderer，查看它的实现，可以发现：

javascript
function ensureRenderer() {
  return (
    renderer ||
    (renderer = createRenderer<Node, Element | ShadowRoot>(rendererOptions))
  )
}

它触发了我们导出的 createRenderer，并且传递了一个 rendererOptions 参数，

javascript
const rendererOptions = /*#__PURE__*/ extend({ patchProp }, nodeOps)

可以发现 rendererOptions 就是一个合并了 patchProp 和 nodeOps 的对象。

这样，我们就成功的把 options 参数传递到了 createRenderer 函数中，createRender函数又调用了 baseCreateRenderer，然后 baseCreateRenderer 函数执行，最终返回一个对象，

实现 ​

  1. 创建 packages/runtime-dom/src/index.ts：
javascript
import { createRenderer } from '@vue/runtime-core'
import { extend } from '@vue/shared'
import { nodeOps } from './nodeOps'
import { patchProp } from './patchProp'

const rendererOptions = extend({ patchProp }, nodeOps)

let renderer

function ensureRenderer() {
	return renderer || (renderer = createRenderer(rendererOptions))
}

export const render = (...args) => {
	ensureRenderer().render(...args)
}
  1. 在 packages/runtime-core/src/index.ts 中导出 createRenderer
  2. 在 packages/vue/src/index.ts 中导出 render
